# Multilogin X TypeScript SDK Design

**Date**: 2026-03-02
**Status**: Approved
**Approach**: Minimal quick-start

---

## Overview

Generate and publish a TypeScript SDK for the Multilogin X API using the `postman-sdk-gen` tool. The SDK will be a minimal, publishable npm package with universal support (Node.js + browser) using native fetch for HTTP requests.

---

## Project Structure

```
multilogin-sdk/
├── Multilogin X API.postman_collection.json  (existing source)
├── package.json                                 (npm metadata)
├── README.md                                   (usage documentation)
└── src/                                         (generated by postman-sdk-gen)
    ├── MultiloginClient.ts                      (main client class)
    ├── launcher/                                (namespace)
    ├── profileAccessManagement/                  (namespace)
    ├── profileManagement/                       (namespace)
    ├── browserProfileData/                      (namespace)
    ├── proxy/                                   (namespace)
    ├── premadeCookies/                          (namespace)
    ├── scriptRunner/                            (namespace)
    └── profileImportExport/                     (namespace)
```

The `src/` directory and all namespace folders are generated automatically from the Postman collection, with one namespace folder per collection folder.

## Package Configuration

### package.json

```json
{
  "name": "multilogin-x-sdk",
  "version": "1.0.0",
  "description": "TypeScript SDK for Multilogin X API - generated from Postman collection",
  "type": "module",
  "main": "./src/MultiloginClient.ts",
  "types": "./src/MultiloginClient.ts",
  "exports": {
    ".": {
      "types": "./src/MultiloginClient.ts",
      "default": "./src/MultiloginClient.ts"
    }
  },
  "files": [
    "src",
    "README.md"
  ],
  "keywords": ["multilogin", "sdk", "api", "typescript", "browser-profiles", "automation"],
  "author": "",
  "license": "MIT",
  "engines": {
    "node": ">=18.0.0"
  },
  "devDependencies": {
    "@sirhc77/postman-sdk-gen": "^0.2.0",
    "typescript": "^5.0.0"
  }
}
```

**Key design decisions:**
- **type: "module"**: ES modules for modern Node.js (18+)
- **Direct TS sources**: No build step - publish TypeScript sources directly for minimal setup
- **Node 18+ requirement**: Required for native fetch support
- **Zero runtime dependencies**: Uses native fetch only

## Generation Workflow

### 1. Install SDK generator

```bash
npm install -g @sirhc77/postman-sdk-gen
```

### 2. Generate SDK

```bash
postman-sdk-gen "Multilogin X API.postman_collection.json" \
  --output ./src \
  --client-name MultiloginClient \
  --use-fetch
```

**CLI flags:**
- `--output ./src`: Output to src/ directory
- `--client-name MultiloginClient`: Main class name (default: ApiClient)
- `--use-fetch`: Use native fetch instead of Axios

### 3. Publish to npm

```bash
npm publish
```

## Usage Example

```typescript
import { MultiloginClient } from 'multilogin-x-sdk';

const client = new MultiloginClient({
  baseUrl: 'https://api.multilogin.com',
  // Auth configured from Postman auth scheme
});

// Namespace-based endpoint access
const profiles = await client.profileManagement.list();
const profile = await client.profileManagement.create({
  name: 'My Profile',
  browserType: 'chrome'
});
await client.launcher.startProfile(profile.id);
```

## Generated Namespaces

The SDK provides 8 namespace modules, one per Postman collection folder:

| Namespace | Purpose |
|-----------|---------|
| `launcher` | Start, stop, and query browser profiles |
| `profileAccessManagement` | Sign in, manage tokens and workspaces |
| `profileManagement` | Create, update, delete browser profiles |
| `browserProfileData` | Unlock browser profiles |
| `proxy` | Proxy configuration and management |
| `premadeCookies` | Pre-made cookie management |
| `scriptRunner` | Script execution and management |
| `profileImportExport` | Profile import and export |

---

## Authentication

The SDK automatically supports authentication methods defined in the Postman collection:

- **Bearer Token** (recommended for automation tokens)
- **API Key**
- **Basic Auth**

Users obtain tokens via Multilogin's token endpoints. Regular tokens have 30-minute lifetime; automation tokens have higher rate limits and longer lifetime.

## Out of Scope

For this minimal quick-start approach, the following are explicitly excluded:

- Build pipeline (no compilation, no bundler)
- CI/CD automation
- Automated tests
- Version management automation
- Separate .d.ts generation (inline types used)
- CommonJS/UMD output (ES modules only)
- Development tooling beyond basic TypeScript

---

## Alternatives Considered

| Approach | Pros | Cons | Decision |
|----------|------|------|----------|
| **Minimal (chosen)** | Fastest path, minimal config, publishable immediately | No build step, users compile TS themselves | ✅ Selected for speed |
| Standard | Build with tsup, proper package.json, basic tests | More setup time (~2-3 hours) | Too complex for current need |
| Production-grade | CI/CD, automated versioning, full testing | 4-6 hours setup | Overkill for now |
